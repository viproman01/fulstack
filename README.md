# Econo Engine

Econo Engine — это веб-приложение для эконометрического анализа данных. Платформа позволяет пользователям загружать наборы данных (CSV-файлы), получать базовую описательную статистику, визуализировать матрицу корреляций и строить модели линейной регрессии. 

Проект разработан в рамках технического задания (ТЗ) на позицию Fullstack разработчика.

## 🚀 Реализованный функционал согласно ТЗ

1. **Загрузка данных:**
   - Поддержка форматов `.csv` файлов.
   - Валидация входных данных: обработка пропущенных значений (NaN) и нечисловых данных, фильтрация таблиц только по числовым колонкам.

2. **Описательная статистика:**
   - Вычисление базовых метрик для каждого признака: Минимум (Min), Максимум (Max), Среднее значение (Mean), Стандартное отклонение (Std).

3. **Корреляционный анализ:**
   - Построение матрицы корреляций Пирсона для оценки мультиколлинеарности.
   - Визуализация матрицы корреляций с помощью интерактивного Heatmap графика.

4. **Линейная регрессия:**
   - Выбор целевой переменной (Target) и независимых переменных (Features).
   - Обучение модели Multiple Linear Regression (с использованием библиотеки `statsmodels`, метод OLS).
   - Вывод детального эконометрического отчета (R-squared, Adjusted R-squared, F-statistic, коэффициенты, P-values и стандартные ошибки).
   - Проверка гипотез: выявление статистически значимых предикторов (p > |t|).

## 🛠 Стек технологий

**Backend:**
- **Python 3.11+**
- **FastAPI** — высокопроизводительный современный веб-фреймворк для построения API.
- **Pandas** — для обработки и очистки данных.
- **Statsmodels** — для проведения эконометрических расчетов и OLS-регрессии.
- **Uvicorn** — ASGI-сервер.

**Frontend:**
- **React 18** — библиотека для создания пользовательских интерфейсов.
- **TypeScript** — строгая типизация для предотвращения ошибок на этапе компиляции.
- **Vite** — сверхбыстрый сборщик модулей.
- **Plotly.js (react-plotly.js)** — интерактивные графики для визуализации данных.
- **Tailwind CSS / CSS Modules** (или Vanilla CSS, зависящий от реализации) — для адаптивной и современной верстки.

**Инфраструктура:**
- **Docker** & **Docker Compose** — для контейнеризации и удобного развертывания проекта "в один клик".

## ⚙️ Как запустить проект

### 🐳 Запуск через Docker Compose (Рекомендуется)

Это самый простой и надежный способ запустить всё приложение со всеми зависимостями.

1. Убедитесь, что у вас установлен Docker и Docker Compose.
2. Клонируйте репозиторий и перейдите в папку проекта:
   ```bash
   git clone <URL_РЕПОЗИТОРИЯ>
   cd econo-engine
   ```
3. Выполните команду запуска в терминале:
   ```bash
   docker-compose up --build
   ```
4. После успешного запуска сервисы будут доступны по следующим адресам:
   - **Frontend (Пользовательский интерфейс):** [http://localhost:80](http://localhost:80)
   - **Backend API:** [http://localhost:8000](http://localhost:8000)
   - **Swagger UI (Интерактивная дока API):** [http://localhost:8000/docs](http://localhost:8000/docs)

### 💻 Локальный запуск (Без Docker)

**1. Запуск Backend сервиса:**
```bash
cd backend
python3 -m venv venv
source venv/bin/activate  # Для Windows: venv\Scripts\activate
pip install -r requirements.txt
uvicorn main:app --reload --host 0.0.0.0 --port 8000
```

**2. Запуск Frontend сервиса:**
Откройте новое окно терминала:
```bash
cd frontend
npm install
npm run dev
```
Фронтенд будет доступен по локальному адресу Vite (обычно http://localhost:5173).

## 📊 Инструкция по тестированию платформы

Для проверки работы приложения подготовлен тестовый набор данных — классический датасет **Iris**:
- Вы можете использовать файл `iris.csv`, который находится в корне проекта.
- **Шаг 1:** В веб-приложении загрузите файл `iris.csv`.
- **Шаг 2:** Просмотрите сгенерированную таблицу с описательной статистикой (все 4 числовые колонки: `SepalLengthCm`, `SepalWidthCm`, `PetalLengthCm`, `PetalWidthCm`).
- **Шаг 3:** Перейдите к матрице корреляций, чтобы визуально определить зависимости (например, высокую корреляцию между `PetalLengthCm` и `PetalWidthCm`).
- **Шаг 4:** В секции "Линейная регрессия" выберите целевую переменную (Target), например, `PetalWidthCm`, а в качестве предикторов (Features) выберите `PetalLengthCm` и `SepalLengthCm`.
- **Шаг 5:** Нажмите "Вычислить" и проанализируйте полученные OLS-результаты (коэффициент детерминации \( R^2 \) и p-value для каждого признака).
